Debian 9安装steam

Posted on 2017-07-16 | Edited on 2018-05-18

计算机环境

  • OS: Debian 9.0 stretch 64bit
  • CPU: Intel Core i7-6700HQ CPU @ 3.5GHz
  • GPU: Mesa DRI Intel(R) HD Graphics 530 (Skylake GT2)
  • 机械革命X6ti

假期在家以折腾为主，steam的安装尝试了不少次，但是因为每次都作死要去安装NVIDIA的闭源驱动，结果每次都搞挂掉。所以这里给出一个忠告，日常使用尽量不要去用英伟达的闭源驱动
了，坑很多，还学不到啥东西。如果确实想要使用（不折腾会死星人 tx），推荐使用Ubuntu，Linux Mint等基于Ubuntu的发行版，它们的附加驱动管理非常好用。下面切入正题，安装
steam。

  • 首先进入steam官网下载steam的deb包。
  • 然后使用cd命令进入所在的文件夹，使用dpkg安装deb包。
    sudo dpkg --install xxx.deb
    此时会出现报错，提示缺少依赖python-apt,手动安装依赖。
    sudo apt install python-apt,再次使用dpkg安装，安装完成。
  • 启动steam，这时会发现启动失败，提示缺少32位库libc.so.6，经过查阅资料手动安装libc6-i386
    sudo apt install libc6:i386。
  • 再次启动steam，在进行初始更新后,程序依然不能够正常运行，提示缺少libGL.so.1，与上文一样，查找资料后手动安装libgl1-mesa-glx:i386。
    sudo apt install libgl1-mesa-glx:i386。再次运行，大功告成。可以去愉快地喜加一了。

备注

由于笔者并没有安装NVIDIA的闭源驱动，只能够使用Intel的核显，所以并没有安装大型的游戏进行测试，如果使用了闭源驱动，有些依赖需要安装的东西是不相同的，这一点需要注意。另
外网络上的资料需要甄别，具体情况具体对待，不同的机器或者发行版差别还是挺大的。如果不熟悉闭源驱动的安装，上网查阅资料的时候一定要注意NVIDIA关键字，不要抱着侥幸心理，>误装了驱动系统崩掉了就不好玩了，笔者在这上面吃过亏。

后记

这次折腾主要的问题出现在32bit和64bit上，目前对于这个知识点的了解并不充分，后期来补上。arch能够果断弃用i386还是很用勇气的，debian还是相对更保守一点。
