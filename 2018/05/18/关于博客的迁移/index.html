关于博客的迁移

Posted on 2018-05-18 | Edited on 2018-05-19 | Views:

陆陆续续地使用过不少博客系统,主流的诸如 Hexo,wordpress,Hugo 之类的都用过.当时选择这些的理由无非是什么速度快啊,插件多啊,现在觉得还是想要去玩的心态比较多.博客系统最主要的功能应该还是写博客,其他的都是次要的.看看这更新频率着实有些汗颜…

个人使用博客主要经历了如下几个阶段:

  • Github Pages + Cloud flare
  • 腾讯云 VPS 自建 (使用了 Hexo)因为没有备案,配置 SSL 证书之类的相当麻烦,故放弃.
  • GCE VPS 自建(使用了 Hugo) 续费续不起…
  • Github Pages

之所以用回来一个是已经习惯了 Hexo 这套操作,没有什么使用成本,能够专注地进行博文写作.还有就是 CF 的加成还是比较厉害的,访问速度算是能够接受,总体体验良好.

恢复博客遇到了一些问题,用了一段时间来解决.

  • 备份问题,我之前是直接用西数的备份工具备份到移动硬盘里的,本地一个备份用的文件夹里并没有,找了一会儿才找到(被自己蠢哭
  • 备份文件直接覆盖之后遇到了字符编码问题,怀疑是 Windows 的 Bug,通过直接传输 Markdown 文件至树莓派中解决.
PS

个人是一个玩心比较重的人,可是好奇心也很重要吧,这样才有动力不断去学习新的东西.
